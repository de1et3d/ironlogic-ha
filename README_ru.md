Используйте на свой риск.

## 📌 Возможности

- Удалённое управление замком - открытие из HA (мгновенно через HTTP API + очередь Web-JSON)
- События в реальном времени - отслеживание проходов, статуса двери, команд по сети
- Управление ключами - добавление, удаление, импорт/экспорт прямо из HA
- Датчик двери - бинарный сенсор положения двери (требуется геркон, включается переключателем)
- Мониторинг доступности контроллера - все сущности становятся недоступными при отключении контроллера
- Перезагрузка контроллера - перезагрузка контроллера прямо из HA
- Локальная работа - контроллер автономен даже при отключении HA
- Несколько контроллеров - поддержка нескольких устройств
- Управление настройками - включение датчика двери и интервал опроса прямо на странице устройства
- Локализация - русский и английский языки

## 🧪 Тестовая среда

Интеграция тестировалась на следующем оборудовании и версиях:

| Компонент | Версия |
| :--- | :--- |
| **Контроллер** | IronLogic Z-5R (мод. Wi-Fi) |
| **Версия ПО контроллера** | 2.55 |
| **Версия ПО модуля связи** | 1.79 |
| **Режим Web-JSON** | HTTP (не Websocket) |
| **Home Assistant** | Core 2026.3.3, 2026.3.4 (HA OS) |
| **Supervisor** | 2026.03.2 |
| **Операционная система** | 17.1 |
| **Frontend** | 20260312.0 |

**Важные замечания:**
- HTTPS может работать нестабильно или не работать вовсе. Для связи с контроллером используйте HTTP.
- Если нужен HTTPS для HA, а контроллеру нужен HTTP, используйте обратный прокси (например, аддон Nginx Proxy Manager).
- Контроллер и HA должны находиться в одной подсети. Работа в разных подсетях не тестировалась.
- Доменные имена можно использовать, если контроллер имеет доступ к DNS.

## 📦 Установка

### Через HACS (рекомендуется)

1. Откройте HACS в Home Assistant
2. Перейдите в **Интеграции → Пользовательские репозитории**
3. Добавьте репозиторий: `https://github.com/de1et3d/ironlogic-ha`
4. Выберите категорию: **Интеграция**
5. Нажмите **Загрузить** и перезапустите HA

### Ручная установка

1. Скачайте последний релиз
2. Скопируйте `custom_components/ironlogic` в `config/custom_components/`
3. Перезапустите Home Assistant

## ⚙️ Настройка

### Порядок установки

1. **Добавьте интеграцию**
   - Перейдите в **Настройки → Устройства и службы → Добавить интеграцию**
   - Найдите **"IronLogic IP Controller"**
   - Выберите способ настройки:
     - **Найти контроллеры автоматически** — сканирует локальную подсеть /24 (до 254 IP-адресов) в поисках контроллеров IronLogic
     - **Настроить вручную** — введите IP-адрес вручную

2. **Если выбран автоматический поиск:**
   - Дождитесь окончания сканирования (до 30 секунд)
   - Выберите контроллер из списка
   - Введите **Имя пользователя** и **Ключ аутентификации**

3. **Если выбрана ручная настройка:**
   - Введите:
     - **Host** - IP‑адрес контроллера (например, `192.168.1.100`)
     - **Username** - зависит от модели: Z-5R (Wi-Fi): `z5rwifi`, Z-5R Web: `z5rweb`, Matrix-II (Wi-Fi): `matrix`
     - **Auth Key** - 8‑символьный ключ с контроллера. Если вы меняли заводской, найдите его в настройках контроллера в разделе "Режим работы".

4. **Настройте вебхук**
   - На странице устройства нажмите кнопку **"Установить webhook URL"**.
   - Затем нажмите **"Перезагрузить контроллер"**, чтобы применить настройки.
   - Контроллер начнёт отправлять события в Home Assistant.

   *При необходимости URL вебхука можно найти в сенсоре **Webhook URL** в разделе Диагностика. Нажмите на него → откройте карточку → три точки → детали → скопируйте атрибут `full_url`.*

5. **Проверьте**
   - После перезагрузки контроллер должен отправить `power_on` в HA.
   - Название устройства обновится на `IronLogic (SN @ IP)` после получения серийного номера.
   - Если ничего не происходит, проверьте сетевую связность и логи.

> ⚠️ **Важно:** Контроллер должен иметь доступ к Home Assistant. Если они в одной сети, используйте локальный IP.

### Отладочный сервер (для диагностики)

Если нужно отладить Web-JSON, используйте встроенный отладочный сервер.
См. файл [debug_server.py](debug_server.py) - простой Python-скрипт, который логирует все запросы контроллера.

## 🎯 События для автоматизаций

| Событие | Описание |
|---|---|
| `ironlogic_key_granted` | Ключ принят, дверь открыта |
| `ironlogic_key_denied` | Ключ отклонён |
| `ironlogic_door_opened` | Дверь открыта (если включён датчик двери) |
| `ironlogic_door_closed` | Дверь закрыта (если включён датчик двери) |
| `ironlogic_door_tampered` | Дверь открыта без авторизации (взлом) |
| `ironlogic_door_left_open` | Дверь оставлена открытой (таймаут) |
| `ironlogic_door_opened_remotely` | Дверь открыта удалённо (из HA или сети) |

### Пример автоматизации

```yaml
automation:
  - alias: "Уведомление при открытии двери ключом"
    trigger:
      - platform: event
        event_type: ironlogic_key_granted
    action:
      - service: notify.mobile_app_phone
        data:
          message: "Дверь открыта ключом {{ trigger.event.data.key_name or trigger.event.data.key }}"

  - alias: "Уведомление о попытке несанкционированного доступа"
    trigger:
      - platform: event
        event_type: ironlogic_key_denied
    action:
      - service: notify.mobile_app_phone
        data:
          message: "Несанкционированная попытка доступа с ключом {{ trigger.event.data.key }}"
```

## 🧠 Как это работает

- **Web‑JSON** - контроллер отправляет POST‑запросы на вебхук HA; HA отвечает командами
- **HTTP API** - HA может мгновенно открыть замок, не дожидаясь следующего `ping` контроллера
- **Мониторинг доступности** - HA периодически проверяет связь с контроллером; при отключении все сущности становятся недоступными
- **Управление настройками** - датчик двери и интервал опроса настраиваются прямо на странице устройства
- Команды ставятся в очередь, если контроллер недоступен - будут доставлены при восстановлении связи

## 🐛 Устранение проблем

### Нет событий от контроллера

- Убедитесь, что контроллер в режиме **Web‑JSON** с протоколом **HTTP**
- Проверьте, что URL сервера указан верно (попробуйте использовать локальный IP, не домен)
- Проверьте сетевую доступность контроллера и HA
- Проверьте логи HA (`custom_components.ironlogic`)

### Ошибка аутентификации

- Проверьте имя пользователя (зависит от модели, чувствительно к регистру)
- Проверьте ключ аутентификации (8 символов, чувствителен к регистру). Если вы меняли его, найдите в веб‑интерфейсе контроллера в разделе "Режим работы".
- Попробуйте зайти в веб‑интерфейс контроллера с теми же данными

### Замок не открывается

- Проверьте проводку между контроллером и замком
- Проверьте настройки типа замка в веб‑интерфейсе контроллера
- Попробуйте открыть замок через веб‑интерфейс контроллера, чтобы исключить проблемы с HA

### Датчик двери не отображается

- Убедитесь, что геркон подключён к контроллеру
- Включите датчик двери через переключатель в Controls устройства
- События двери будут отображаться только при включённом переключателе
